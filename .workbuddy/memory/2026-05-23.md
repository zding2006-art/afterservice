# 2026-05-23

## 售后分析系统：修复多月份拆分与覆盖提示

### Bug 1：月.日 格式日期未正确解析（根因）
- 文件中日期格式为 `9.1`, `10.28`（月.日 小数），旧代码 `int(float(val))` 只取整数部分，导致 `9.1` → 第9天，全归入文件名月份，10月数据被忽略
- **修复**：`normalize_date()` 重写，将 float 值拆分为整数部分（月）和小数百分之（日）：`m=int(fval), d=round((fval-m)*100)`，若 m 在 1-12 且 d 在 1-31 则识别为 月.日 格式

### Bug 2：静默覆盖已存在月份
- 旧逻辑：单月模式直接 `save_monthly()` 不检查，多月模式只是 `skipped` 不提示
- **修复**：
  - `app.py` 单月：先查已存在，返回 `already_exists: True`，不保存
  - `app.py` 多月：返回 `already_exists_months` 列表
  - 新增 `/api/overwrite` 接口，接收前端确认后再覆盖
  - 前端：单月弹出确认对话框，多月弹出列表式确认对话框，用户主动选择"覆盖"或"保留"

### 验证结果
- 文件 202509售后配件记录.xlsx（含9+10月）：正确拆分 9月1150条/10月793条
- 已存在月份不再静默覆盖，弹出确认框
