## V1.6 文件名日期解析修复

### Bug：`2025年6月售后配件记录.xlsx` 被识别为 `2026年5月`
- 根因：`parse_month_from_filename` 的 regex `(\d{4})(\d{2})` 仅匹配"4位数字紧接2位数字"格式
- `2025年6月` 中间有中文字符，regex 无匹配 → fallback 到 `datetime.now()` → 返回当前日期
- 修复：按优先级依次匹配多种格式
  1. `(\d{4})年(\d{1,2})月` → "2025年6月"
  2. `(\d{4})[-/](\d{1,2})` → "2025-06" / "2025/6"
  3. `(\d{4})(\d{2})` → "202506" 紧凑格式

### Excel 导出旧列名修正
- `app.py` Excel 导出 headers 中 `"5年(49月+)"` → `"大于4年"`
