## 2026-06-04 修复 Failed to fetch 登录问题

### 问题
日文部署版登录时报 "パスワードが間違っています。再入力してください。: Failed to fetch"
根因：前端 catch(e) 把 fetch 网络错误拼接到了 login.error 文字后面显示

### 修复内容
1. **start.sh bug 修复**：移除无效的 `--port ${PORT}` 参数（app.py 不接受命令行参数，端口通过环境变量 PORT 传递）
2. **CORS 配置改进**（backend/app.py + deploy/backend/app.py）：
   - `CORS_ORIGINS=""` 时默认允许 localhost/127.0.0.1 各端口
   - `CORS_ORIGINS="*"` 时允许所有来源（调试模式，supports_credentials=False）
   - `CORS_ORIGINS="https://xxx.com"` 时精确指定
3. **start.sh 自动检测本机 IP**：启动时自动添加到 CORS_ORIGINS
4. **deploy/frontend 同步**：复制最新 index.html 和 i18n.js 到 deploy 包
5. **README_JP.md 补充**：添加 "Failed to fetch" 完整排查步骤（日文）

### 关键说明
通过 Nginx 反代或直接 IP:PORT 访问时不会产生 CORS，Flask 同时提供前端和 API，同源。
CORS 问题只在本地开发（file://协议或不同端口）时才出现。

## 2026-06-04 修复前端错误提示混淆 bug

### 问题
`doLogin()` catch 块中把网络错误（Failed to fetch）拼接在 `login.error`（密码错误文字）后面，
误导用户以为是密码错误，实际是后端服务不可达。

### 修复
1. **i18n.js** - 三种语言分别新增 `login.networkError` key：
   - zh-CN: 无法连接到服务器，请确认后端服务已启动
   - en: Cannot connect to server. Please ensure the backend service is running.
   - ja: サーバーに接続できません。バックエンドサービスが起動しているか確認してください。
2. **index.html** - `doLogin()` catch 块改为 `I18N.t('login.networkError')`，不再拼接原始 JS 错误信息
3. 同步到 deploy/frontend 目录
